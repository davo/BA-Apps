Changelog

  • Version 3.0 (Download version 3.0.2)
    1. Simplified sizing in the CSS framework.
    2. Improved styling open-links.
    3. Renamed all namespaces, classname-prefixes and data-prefixes from "mmenu" to "mm".
    4. Added one page scrolling example to the download pack.
    5. Added jQuery Mobile example to the download pack.
    6. Added options for dragging/swiping the menu open.
    7. Improved behavior for the onClick options object.
    8. Fixed bug with cloning the menu.
    9. Added options onClick.setSelected, onClick.setLocationHref and onClick.callback.
    10. Changed the onClick.delayPageLoad option to onClick.delayLocationHref.
    11. Added setPage custom event to change the "page" node.
    12. Added setSelected custom event (on LIs) to change the "selected" menu item.
    13. A numeric value for the onClick.delayLocationHref option is deprecated.
    14. Changed the configuration.slideDuration option to configuration.transitionDuration.
    15. Added option configuration.pageSelector to better specify the "page" node.
  • Version 2.2 (Download version 2.2.3)
    1. Added Opera vendor prefixes to the CSS.
    2. Fixed border-top issue on first search results.
    3. Improved showing labels in search results.
    4. Fixed bug in opening and closing submenus.
    5. Added mmenu-sizing-large.css file for large buttons.
    6. Added option configuration.clone to use a clone of the original menu.
    7. Fixed IE10 a:active issue.
    8. Added $.fn.mmenu.support.transition (only detects default support, no vendor-prefixes).
    9. Replaced timeouts with transitionend (if supported as tested by $.fn.mmenu.support.transition).
    10. Improved re-creating the plugin.
  • Version 2.1 (Download version 2.1.1)
    1. Added configuration.preventTabbing option to prevent tabbing to elements outside the menu when the menu is opened.
    2. Rewrote some code.
    3. Added an "iconbar" example to the download pack.
  • Version 2.0 (Download version 2.0.2)
    1. Fixed scrolling bug on Android 2.3
      Browsers that do not support the CSS property "overflow-scrolling" (on a touch-device) now use a slightly different approach for scrolling through the menu. Trigger this global function to GET or SET whether or not to use this fallback:
      $.fn.mmenu.useOverflowScrollingFallback( [true/false] );
    2. Made the menu themeable by using only one background color and semi-transparent black and white shades.
    3. Added option to open the menu from the top or bottom.
    4. Disabling hardware acceleration is no longer global for the entire page.
    5. Added opening, opened, closing and closed (callback) custom events.
    6. Added reset and search custom events.
    7. Improved CSS.
    8. Improved positioning the page using scrollTop.
    9. Added documentation for custom events to the download pack.
    10. Added global settings for feature detection:
      $.fn.mmenu.support.touch
      $.fn.mmenu.support.overflowscrolling
  • Version 1.4 (Download version 1.4.1)
    1. Added widescreen CSS file for an always opened menu on wider screens.
    2. Fixed bug with fixed header example.
    3. Small CSS changes for better support.
  • Version 1.3 (Download version 1.3.1)
    1. Changed the closeOnClick option to the onClick object.
    2. Added a minified version to the download pack.
    3. Fixed bug with multiple menus with submenus.
    4. Added default support for searchfields.
    5. Fixed bug when resizing the window if the menu is opened.
    6. Fixed bug with the hash not changing after clicking a link in the menu.
    7. Fixed bug with page jumping back to the top when opening the menu.
    8. Added option configuration.hardwareAcceleration to force hardware acceleration.
    9. Changed the addCounters option to the counters object.
    10. Added documentation to the download pack.
  • Version 1.2 (Download version 1.2.3)
    1. Added three color themes to the examples in the download pack.
    2. Added support for pre-defined counters.
    3. Fixed bug with vertical submenus not closing when selected.
    4. Added option addCounters for counters.
    5. Set the robots-metatag in the examples to "noindex,nofollow".
  • Version 1.1 (Download version 1.1.0)
    1. Added two color themes to the examples in the download pack.
    2. Added support for labels.
    3. Added option closeOnClick.
  • Version 1.0 (Download version 1.0.2)
    1. Fixed bug with vertical submenu in menu on the right.
    2. First release of the plugin.
